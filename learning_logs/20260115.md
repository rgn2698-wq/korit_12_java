## catch 상에서의 sout vs throw new
= catch 블록 내에서 sout을 쓰는것과 다시 예외를 던지는 throw new 것의 차이는 내부에서 문제를 해결한것은가 vs 상위 메서드에게 보고를 할 것인가 라고 비유 할수있음.

1.System.out.println() - 예외처리후 소멸
    - 여기서 문제해결을 다 헀고 프로그램을 진행하라는 의미.
        1. 목적: 사용자에게 에러르 안내하고 프로그램이 일비정상 종료되지 않게 처리하는것.
        2. 특징 : catch문 이후에 추가적인 코드들이 직접적으로

```java
try {
    int result = 10 / 0;
} catch (AritimeticException e) {
    System.out.println("0으로 나눌수 없습니다.");   // 이거 띄워놓고
}
System.out.printLn("ㅡㅡㅡ");   // catch 배열의 코드가실행
```

2. throw new (예외 재발생 = 전파)
- 예외 처리 자체가 불가능. 혹은 해당 사실을 상위 메서드에 전달해줘야 한다는 것.
    1. 목적 : 예외르르 _다시_ 발생시켜 상위 메서드로 예외를 전달 하는것.
    2. 현재 메서드는 즉시 중단되며, 상위 메서드에서 다시 try,catch로 잡아내거나 또 throw로 던지는 상태
```java
try {
    int result = 10 / 0;
} catch (AritimeticException e) {
    System.out.println("0으로 나눌수 없습니다.");   // 이거 띄워놓고
    throw new RuntimeException("~~~") 
}
System.out.printLn("ㅡㅡㅡ");
```
